Detailed Description

template<std::size_t N, typename Index>
class ph::math::TWideBvhNode< N, Index >
Note
The layout of this node is different to ordinary node layout–children are directly stored in their parent, resulting in a "fat" node.

The most straightforward interpretation of the split axis is the axis that this child (childIdx) and next child (childIdx + 1) is divided upon. However, some tree types may store the split axis in a way such that the axis associated with childIdx is interpreted differently for efficiency reasons.

Returns

Alignment in bytes for the SoA data view. The alignment is intended for use with SIMD computing. Note that additional alignment for double is generally not needed. For example, AVX for 8-wide packed double requires only 32-byte alignment.
