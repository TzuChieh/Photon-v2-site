Calculates the area extended by this primitive. 

The term "{}extended"{} implies single-\/sided, e.\+g., a triangle\textquotesingle{}s extended area is half the absolute value of the cross product of its two edge vectors. To treat it as double-\/sided, you need to multiply the result by 2 manually.

A zero return value means the concept of extended area does not apply to this primitive.

Determine whether a given ray hits the object. 

Checks whether the specified ray intersects this intersectable. If there is an intersection, true is returned and a brief hit report is stored inside the probe. If there is no intersection, false is returned and the state of the probe is undefined. {\ttfamily ray} and {\ttfamily probe} can be used for obtaining hit detail if an intersection is found.

bool ph\+::\+PTriangle\+::may\+Overlap\+Volume (\begin{DoxyParamCaption}\item[{const \mbox{\hyperlink{namespaceph_1_1math_a79409e9e8aea24d984f98f3acb483937}{math\+::\+AABB3D}} \&}]{volume}{}\end{DoxyParamCaption}) const\hspace{0.3cm}{\ttfamily [override]}, {\ttfamily [virtual]}}



Conservatively checks whether this object overlaps a volume. 

By conservative, it means {\bfseries{true can be returned even though the object does not overlap the volume}}; but if it actually does, {\bfseries{true must be returned}}. The default implementation performs conservative intersecting test using the AABB calculated by \doxylink{classph_1_1_p_triangle_ac8552d3dd854599a6b0c74fc07c85a5f}{calc\+AABB()}. Although false-\/positives are allowed for this method, providing an implementation with higher accuracy is benefitial for many algorithms used by the renderer. The test generally considers the underlying shape as {\bfseries{hollow}} (for closed shape), while the volume is {\bfseries{solid}}.

Intersect the intersected object again with a different ray. 

This method is different to is\+Intersecting(const Ray, Hit\+Probe\&) const. Given {\ttfamily src\+Ray} and {\ttfamily src\+Probe}, this method performs an intersection test against the chain of intersectables recorded in {\ttfamily src\+Probe} (so it is impossible to "{}discover"{} new intersectables with this method). Taking BVH as an example, the implementation may only record the intersected object in the probe so this method can bypass the entire BVH traversal.
