\doxysubsection{Detailed Description}
IO functions and helpers for processing EXR files. 

Also contains helpers for using ILM\textquotesingle{}s Open\+EXR library for processing EXR files. The basic idea is that we provide pixel data in {\ttfamily TFrame}, then create {\ttfamily Imf\+::\+Header} and map frame data using {\ttfamily Imf\+::\+Frame\+Buffer}. Finally, pixel data is read/written to file using {\ttfamily Imf\+::\+Input\+File}/{\ttfamily Imf\+::\+Output\+File}. If the element type in {\ttfamily TFrame} does not match the precision we requested, then a conversion will be done (converting to a suitable {\ttfamily TFrame} on the fly, which incurs additional memory allocation).

To simplify common usages, {\ttfamily TFrame} is assumed to contain the full data window. If a display-\/window only {\ttfamily TFrame} is desired, use {\ttfamily adjust\+\_\+frame\+\_\+to\+\_\+display\+\_\+window()}.

Note that {\ttfamily Imf\+::\+Frame\+Buffer} and {\ttfamily Imf\+::\+Slice} do not own the specified data, they only reference it. The original pixel data (typically a {\ttfamily TFrame} here) must be kept alive during the output process.
