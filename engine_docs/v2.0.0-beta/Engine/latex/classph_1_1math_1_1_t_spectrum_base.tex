\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename Derived, \mbox{\hyperlink{namespaceph_1_1math_a957eab3ceed525ebce5c24b6b57eb975}{EColor\+Space}} COLOR\+\_\+\+SPACE, typename T, std\+::size\+\_\+t N$>$\newline
class ph\+::math\+::\+TSpectrum\+Base$<$ Derived, COLOR\+\_\+\+SPACE, T, N $>$}
Base for spectrum implementations. 

An important assumption is that spectrum implementations are all color samples that can be linearly combined. {\ttfamily \doxylink{conceptph_1_1math_1_1_c_color_values_interface}{CColor\+Values\+Interface}} can be used for directly access the raw sample values.

\begin{DoxyNote}{Note}
It is not recommended to add your own field in your spectrum implementation. If you must, make sure there will not be inconsistent object state (slicing) when {\ttfamily CColor\+Transform\+Interface} is being used. 
\end{DoxyNote}
