Define a SDL class with function-\/like syntax. 

Use this macro in the class body of a SDL resource to provide an interface for introspection. The SDL class instance type to be defined should be the only argument to the macro. Followed is a ordinary C++ scope (a pair of curly braces) where the definition for the SDL class instance resides. Several utility types can be used within the scope\+: {\ttfamily Class\+Type} for the type of the SDL class, and {\ttfamily Owner\+Type} for the type of the SDL resource. Finally, use a return statement to return the SDL class instance.

Available functionalities after defining the macro\+:

{\ttfamily const Class\+Type\texorpdfstring{$\ast$}{*} get\+Sdl\+Class()}
\begin{DoxyItemize}
\item A static method for accessing the static SDL class.
\end{DoxyItemize}

{\ttfamily const Sdl\+Class\texorpdfstring{$\ast$}{*} get\+Dynamic\+Sdl\+Class() const}
\begin{DoxyItemize}
\item A virtual method for accessing SDL class in runtime, through an instance to resource. The returned class will be the one defined for the actual type of the resource.

Define a SDL enum with function-\/like syntax. 

One does not need to define all enum entries. It is valid to partially expose some enum entries and keep the others hidden from SDL. Note that it is highly encouraged to provide a valid entry with the value 0 as it will be used as the default.

A limitation of this macro is that it must be called in the enclosing namespace of {\ttfamily ph} (within namespace {\ttfamily ph} itself). Note this should not cause any name collision--the enum type can still be defined in an appropriate namespace. (The underlying implementation depends on explicit template specialization which requires the aforementioned rule. As long as a unique enum type is used for specializing {\ttfamily TSdl\+Enum}, the resulting type should still be unique.)

For a enum {\ttfamily Enum\+Type}, after the definition is done by calling this macro, you can use methods in {\ttfamily TSdl\+Enum\texorpdfstring{$<$}{<}Enum\+Type\texorpdfstring{$>$}{>}} to access functionalities.
