◆ nextTimestampedCount()

uint64 ph::editor::IDGenerator::nextTimestampedCount ( )
static

Similar to nextCount(), with a time point associated to it. This variant offers better chance that the ID is unique across a program's lifetime. The uniqueness of the ID can fail if:

  1. The program generates new IDs in a tight loop (more IDs than the number of time units between two consecutive program startup);
  2. User adjusts system time. In the worst case this method degenerates to nextCount(). Never treat this as UUID.
    Note
    Thread safe.
