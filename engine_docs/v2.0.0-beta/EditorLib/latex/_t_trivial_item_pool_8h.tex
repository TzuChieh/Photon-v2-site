class \mbox{\hyperlink{classph_1_1editor_1_1_t_trivial_item_pool}{ph\+::editor\+::\+TTrivial\+Item\+Pool$<$ Item, Dispatcher $>$}}
\begin{DoxyCompactList}\small\item\em Item pool for simple types. This item pool is designed to minimize execution time and memory footprint. As a result, some operations are not possible compare to {\ttfamily \doxylink{classph_1_1editor_1_1_t_item_pool}{TItem\+Pool}}, e.\+g., iterating over created items, clear, etc. User is expected to keep track of the handles and use them to iterate the container the way they prefer. When using manual handle management APIs, lost handles cannot be retrieved and those storage slots are effectively leaked (pool dtor will still correctly free the memory though). \end{DoxyCompactList}\end{DoxyCompactItemize}
