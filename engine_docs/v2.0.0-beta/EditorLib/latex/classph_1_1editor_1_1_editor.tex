\doxylink{classph_1_1editor_1_1_editor}{Editor} events. Subscribe to editor event by adding listener to the corresponding dispatcher. Note that when listening to events from a non-\/permanent resource, be sure to unsubscribe to the event before the resource is cleaned up so the event system will not act on a dangling resource. Unsubscribing to an event can be done by removing listeners from the dispatcher, e.\+g., by calling {\ttfamily \doxylink{classph_1_1editor_1_1_t_class_event_dispatcher_a0ef23f1eeb37e93f0709698caa5fe684}{TEditor\+Event\+Dispatcher\+::remove\+Listener\+Immediately()}} (calling the non-\/immediate overload is also acceptable). 

\doxylink{classph_1_1editor_1_1_editor}{Editor} events can also be submitted in a delayed/buffered manner, and with thread safety guarantee. To submit event like this, use {\ttfamily \doxylink{classph_1_1editor_1_1_editor_a98d48eaf368f548f883f744fd39198e9}{post\+Event()}}. Do not reference anything that might not live across frames/threads when posting event.
