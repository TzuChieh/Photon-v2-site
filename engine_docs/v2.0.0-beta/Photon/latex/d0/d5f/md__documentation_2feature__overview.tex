A quick overview of the features supported by Photon. Similar to \doxysectlink{md__documentation_2project__structure}{Project Structure}{0}, this page can also serve as a handy reference for you to find the documentation or source code that you need.\hypertarget{md__documentation_2feature__overview_autotoc_md21}{}\doxysection{\texorpdfstring{Visualizer}{Visualizer}}\label{md__documentation_2feature__overview_autotoc_md21}
Rendering is the process of visualizing a scene. In Photon, we use a \textbf{ Visualizer} to generate a \textbf{ Renderer} to actually render an image. This is where most of the core algorithms meet and work together--and is arguably the most complex part of a renderer. While there are many types of renderers to choose from, there is hardly an all-\/purpose renderer as each one of them is designed to deal with different situations. The efficiency of a rendering algorithm is also partly depending on the characteristics of a scene, e.\+g., sizes of geometries, material properties, etc. At the extreme end, one can always construct a scene that will fail a certain rendering algorithm. Be sure to understand and try different renderers if things do not work out.\hypertarget{md__documentation_2feature__overview_bvpt}{}\doxysubsection{\texorpdfstring{Unidirectional Path Tracing}{Unidirectional Path Tracing}}\label{md__documentation_2feature__overview_bvpt}
 
\begin{DoxyImage}
\includegraphics[width=\textwidth,height=\textheight/2,keepaspectratio=true]{055_books_based_on_Libri_sulla_mensola_books_by_archemi.jpg}
\doxyfigcaption{An example image rendered using this renderer.}
\end{DoxyImage}
   

Though this is a relatively simple renderer to solve the {\itshape rendering equation} as proposed by Kajiya \cite{Kajiya:1986:Rendering}, this renderer is a good starting point before diving into other solvers. Backward unidirectional path tracing does not handle small light sources well, and scenes with hard-\/to-\/find light transport paths are likely to fail, too. This renderer can be constructed by creating a \textbf{ Path\+Tracing\+Visualizer} with BVPT energy estimator (use BVPTDL if only direct lighting is required).\hypertarget{md__documentation_2feature__overview_bneept}{}\doxysubsection{\texorpdfstring{Unidirectional Path Tracing with NEE}{Unidirectional Path Tracing with NEE}}\label{md__documentation_2feature__overview_bneept}
 
\begin{DoxyImage}
\includegraphics[width=\textwidth,height=\textheight/2,keepaspectratio=true]{bathroom_based_on_Salle_de_bain_by_nacimus.jpg}
\doxyfigcaption{NEE can handle more complex scenes.}
\end{DoxyImage}
   

With the addition of Next Event Estimation (NEE), unidirectional path tracing can handle common scenes quite effectively. While largely based on the same principle as the non-\/\+NEE variant, this renderer combines additional sampling techniques \cite{Veach:1995:Optimally} to better handle smaller light sources. Similarly, create a \textbf{ Path\+Tracing\+Visualizer} with BNEEPT energy estimator to use this renderer.\hypertarget{md__documentation_2feature__overview_vpm}{}\doxysubsection{\texorpdfstring{Vanilla Photon Mapping}{Vanilla Photon Mapping}}\label{md__documentation_2feature__overview_vpm}
 
\begin{DoxyImage}
\includegraphics[width=\textwidth,height=\textheight/2,keepaspectratio=true]{vpm_perfumes_by_gp3991.jpg}
\doxyfigcaption{Perfume bottle and its cap rendered with 500M photons (radius\+: 0.1, spp\+: 4). The image has not rendered to convergence to better show the characteristics between different photon mapping techniques.}
\end{DoxyImage}
   

It is worth to clarify that the project is named {\itshape Photon} not because the focus is on photon mapping techniques, but because photon is the elementary particle that transmit light, and the core of this project is to solve light transport problems. Nevertheless, several photon mapping techniques are available here, and the most fundamental one is vanilla photon mapping \cite{Jensen:1996:Global}. All photon mapping techniques are capable of {\itshape merging} nearby light transport paths in some way, and this form of path reuse allows them to excel on a wider variety of lighting phenomena. You can get a vanilla photon mapping renderer by creating a \textbf{ Photon\+Mapping\+Visualizer} in Vanilla mode.\hypertarget{md__documentation_2feature__overview_ppm}{}\doxysubsection{\texorpdfstring{Progressive Photon Mapping}{Progressive Photon Mapping}}\label{md__documentation_2feature__overview_ppm}
 
\begin{DoxyImage}
\includegraphics[width=\textwidth,height=\textheight/2,keepaspectratio=true]{ppm_perfumes_by_gp3991.jpg}
\doxyfigcaption{Perfume bottle and its cap rendered with 0.5M photons for 1000 passes (radius\+: 0.1, spp\+: 4). The image has not rendered to convergence to better show the characteristics between different photon mapping techniques.}
\end{DoxyImage}
   

This renderer implements the PPM algorithm by Hachisuka et al. \cite{Hachisuka:2008:Progressive} The first pass traces a fixed set of view vertices then the following iterative process will estimate the incoming energy on those vertices. The rendered result will be closer to ground truth with more view vertices (and the above image is rendered with maximum view path length = 10). The algorithm can leverage effectively infinite amount of photons to render a scene, achieving superior results then vanilla photon mapping. Create a \textbf{ Photon\+Mapping\+Visualizer} in Progressive mode to use this renderer.\hypertarget{md__documentation_2feature__overview_sppm}{}\doxysubsection{\texorpdfstring{Stochastic Progressive Photon Mapping}{Stochastic Progressive Photon Mapping}}\label{md__documentation_2feature__overview_sppm}
 
\begin{DoxyImage}
\includegraphics[width=\textwidth,height=\textheight/2,keepaspectratio=true]{sppm_perfumes_by_gp3991.jpg}
\doxyfigcaption{Perfume bottle and its cap rendered with 0.5M photons for 1000 passes (radius\+: 0.1, spp\+: 4). The image has not rendered to convergence to better show the characteristics between different photon mapping techniques.}
\end{DoxyImage}
   

This renderer implements the SPPM algorithm by Hachisuka et al. \cite{Hachisuka:2009:Stochastic}, and is intended to serve as a reference implementation for other more sophisticated techniques. Tricks and shortcuts are avoided if possible, so the evaluated radiance is more likely to remain correct as the project evolves. Performance is not a major concern for this renderer, so do not use it in situations where time is valuable. Create a \textbf{ Photon\+Mapping\+Visualizer} in Progressive mode to use this renderer.\hypertarget{md__documentation_2feature__overview_pppm}{}\doxysubsection{\texorpdfstring{Probabilistic Progressive Photon Mapping}{Probabilistic Progressive Photon Mapping}}\label{md__documentation_2feature__overview_pppm}
 
\begin{DoxyImage}
\includegraphics[width=\textwidth,height=\textheight/2,keepaspectratio=true]{pppm_perfumes_by_gp3991.jpg}
\doxyfigcaption{Perfume bottle and its cap rendered with 0.5M photons for 1000 passes (radius\+: 0.1, spp\+: 4). The image has not rendered to convergence to better show the characteristics between different photon mapping techniques.}
\end{DoxyImage}
   

This renderer is based on Knaus and Zwicker\textquotesingle{}s paper \cite{Knaus:2011:Progressive}, which decouples any intermediate radiance estimate from previous iterations. We can treat any photon mapping technique as a black box and parallelizes the calculation of each iteration trivially. However, this renderer consumes more memory in comparison with others. Create a \textbf{ Photon\+Mapping\+Visualizer} in Progressive mode to use this renderer.\hypertarget{md__documentation_2feature__overview_autotoc_md22}{}\doxysubsection{\texorpdfstring{Scheduler}{Scheduler}}\label{md__documentation_2feature__overview_autotoc_md22}
Some \textbf{ Visualizer}s can make use of a \textbf{ Work\+Scheduler} to distribute rendering work over multiple processor cores. Different types of schedulers dispatch their work in different ways. When rendering an image, a scheduler will typically use the dimensions of the image and number of iterations to define the total volume of work.

\hypertarget{md__documentation_2feature__overview_autotoc_md23}{}\doxysection{\texorpdfstring{Observer}{Observer}}\label{md__documentation_2feature__overview_autotoc_md23}
\textbf{ Observer} plays an important role in a scene\+: it captures incoming energy and record it on its sensing device called \textbf{ Film}--so you can observe the virtual world which just got rendered. A common example of an observer is camera, which senses and records lighting as a photograph.\hypertarget{md__documentation_2feature__overview_autotoc_md24}{}\doxysubsection{\texorpdfstring{Pinhole Camera}{Pinhole Camera}}\label{md__documentation_2feature__overview_autotoc_md24}
 
\begin{DoxyImage}
\includegraphics[width=\textwidth,height=\textheight/2,keepaspectratio=true]{pinhole_camera_teaser.jpg}
\doxyfigcaption{Image rendered by a pinhole camera. Notice the sharpness in foreground and background; there is no difference and it appears to be in focus everywhere.}
\end{DoxyImage}
   

Photon currently supports two types of perspective camera\+: \textbf{ Pinhole\+Camera} and \textbf{ Thin\+Lens\+Camera}. A pinhole camera is simply composed of a hole (which serves as its lens system) and a film. Images captured by this camera is similar to how a normal human perceives the world but with several simplifications. Due to its simplicity, it is widely adopted in computer graphics industry.\hypertarget{md__documentation_2feature__overview_autotoc_md25}{}\doxysubsection{\texorpdfstring{Thin Lens Camera}{Thin Lens Camera}}\label{md__documentation_2feature__overview_autotoc_md25}
 
\begin{DoxyImage}
\includegraphics[width=\textwidth,height=\textheight/2,keepaspectratio=true]{thinlens_camera_teaser.jpg}
\doxyfigcaption{Image rendered by a thin-\/lens camera. Notice the depth of field effect.}
\end{DoxyImage}
   

For thin lens camera, as its name suggests, the lens system in this camera is assumed to be a single lens with negligible thickness. The biggest advantage of it is that depth of field effects are possible under this model. In the following render, depth of field is achieved by a 72 mm lens focusing on the first metallic monkey.\hypertarget{md__documentation_2feature__overview_autotoc_md26}{}\doxysubsection{\texorpdfstring{Environment Camera}{Environment Camera}}\label{md__documentation_2feature__overview_autotoc_md26}
